# Merge Pull Request

![image](https://hackmd.io/_uploads/BJHXbyhwkx.png)

Github 會幫你偵測有沒有 conflict 產生。如果沒有 conflict + reviewer 批准，就可以 Merge 了。

:::note[如果發生 conflict 會怎樣？]
那就要把 conflict 解決。
可以自己製造一個 conflict 練習一下。
如果可以順利製造一個並且解決，說明你真的學會了。
不過協作中這很容易發生，相信遲早會遇到的。
:::

點擊 `Merge pull request` 按鈕就完成了。
之後會看到一個 `Delete branch` 按鈕，可以把 `misc/gitignore` branch 刪除。
我自己是習慣按下去的，這樣會比較乾淨。
